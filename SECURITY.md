# Security Policy

## Supported Versions

Only the latest development version is currently supported.

## Reporting Vulnerabilities

Please do NOT create a public issue for security vulnerabilities.

Instead:

1. Contact the maintainer.
2. Provide a detailed description.
3. Include reproduction steps if possible.

## Scope

Examples:

* Remote Code Execution
* Privilege Escalation
* Sandbox Escape
* Credential Disclosure
* Bypass of Security Features

## Response

Security reports will be reviewed as quickly as possible.

Thank you for helping keep OTX-Sec secure.
